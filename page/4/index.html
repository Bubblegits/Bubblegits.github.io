落幕

幕布落下的那一刻是时间尺度上的一个边界。夸张的，被审视的已经结束，这时候只有没有完全进入戏剧的观众才能不感到失落。如同大梦初醒。对于我们来说，落幕比起死亡更加可怕，因为死亡至少有可能被当作一种解脱，一种无尽的下坠，而落幕却残忍地把我们从云端扔向海面。你把现实看成是向精神世界进发的一场挑战，落幕的时刻也就是决斗钟声敲响的时刻，你要给这场决斗一个结果，而不是让它流产。 观众们是最为无助的。在落幕之前，他们可以完全忘记自我，可这时他们关注的对象从舞台上回到了自身之中，重担再一次压在了肩上。这个时候最为通常的做法是表现地成熟而坚强，暗示自己刚刚的戏剧不过是现实生活中的小小调剂，说自己“虽不能至，心向往之”，也就算了。如果我们能把精神与现实完全区分开来，那也就不会遭到这么多痛苦，可是我们无法做到，所以转而要求戏剧与现实生活的同构。然而你从戏剧里只能看出一种令人厌恶的生活——实际上根本没有什么让你感到满意的生活方式，可是戏剧仍然对无趣的现实恋恋不舍。 观看一个故事比起参与一个故事多了一个层级。如果说戏剧与生活同构，那么我们也可以说是在观看生活，并且借此摆脱生活。 按：受adove启发，参与一个故事其实反而比观看一个故事多一个层级，因为在参与故事时，我们还可以行动。

探店奥利凡德

“一位挑剔的顾客吧，嗯？不要紧，我想，这里总能找到一款最理想，最完美，最适合你的——让我想想看——哦，有了，怎么会没有呢——非凡的组合，冬青木，凤凰羽毛，十一英寸长。不错，也柔韧。” 哈利接过魔杖，感到指尖突然一热。他把魔杖高举过头，飕的一声向下一挥，划过尘土飞扬的空气，只见魔杖头上像烟花一样迸发出一道红色和金色的火花，跳动的光斑投到四壁上。 海格拍手喝彩，奥利凡德先生大声喊起来：“哦，好极了，哦，真的，太好了。哎呀，哎呀，哎呀……太奇妙了……真是太奇妙了……” 我现在手上的笔记本是六七年前的低端产品，估计是thinkpad L480，里面的赛扬CPU神龟虽寿，犹有竟时，到如今打开三个以上的网页就已经不堪重负，所以是时候换个电脑了。 美国电子产品更新换代的速度完全赶不上国内，上网一搜到处都是两三代之前的电脑，价格还有模有样的。我本来想要一个AMD的高端CPU，不用独显靠核显撑着，这样在重量和续航上就有更大余地，但不知道为什么这样的需求并不多见，美国市场的主流是入门级CPU或者高端CPU配入门级显卡，而且Intel的笔记本多如牛毛，AMD的U实在太少。另外，像amazon, walmart这样的网站总是把具体配置藏着揶着，Best Buy上信息也不是很全面，幸好有几个网站可以用来查找对比。 pcmagazine对笔记本有比较直观的总体评价，比起数据更侧重使用体验。 nanoreview可以用来对比笔记本，会全面总结出双方的优缺点。 notebookcheck这个网站上有针对各种硬件的专门文章，对其表现的分析非常细致，并且在文末还会介绍搭载了这个硬件的笔记本型号，对于需求比较明确的群体来说非常好用。上面也有关于特定机型的整体测评，保持了专业水准。 最后还是在为数不多的AMD笔记本里选择了thinkpad T14 gen4，毕竟原来thinkpad的键盘手感和小红点我都很喜欢，而且 实际上，当然是魔杖在选择它的巫师呢。

10月13日

将近半个月没有动笔，加上刚刚从伯克利回来，应该写长一点，好好总结一下过去的思考。不如先从主权问题开始。之前我谈过认识的统治，这个说法不太准确，应该说是对认识的统治，进一步地，是我对我的认识的统治。我之前太习惯于把认识这样的重要领域交给别人，常常是交给多数人，或者是随便交给见到的什么人，心甘情愿地拾人牙慧而逃避艰巨且关键的认识任务。仔细想想就会发现，这导致我的认识太大略而且浮于表面，以致于摇摆不定而且自以为是。在幼稚的信念坍塌之后，我一度完全放弃了主权，这导致了我的徘徊。我要掌控自身的认识世界，可这里还有一个问题，一个“我在说谎”的悖论。对于认识的自指，也就是反思，反过来攻击我自己。肯定性的权力以及否定性的权力，我失去了前者而交出了后者。 于是我们就谈到选择。为什么我肯定一个选项而否定另外一个选项？大多数情况下，我仓促地选择保守的那一项。选择保守的方面本身并不成为问题，问题是我没有给自己足够的犹豫。犹豫是一个选择里最为温柔动人的部分，甚至比选择的结果还要动人，许多时候我们并不犹豫，只是通过逃避来拖延时间，而后与之前一样无知地作出选择。厌恶犹豫以及缺少主权，这两者共同造成了保守的结局。 我们已经了解到，权力的获得并不是靠声称，而是靠夺取。所以说你要抓住选择中夺权的机遇以夺取权力，而机遇意味着向可能性敞开。选择更具有可能性的一项，并不是因为这一项更好——“好”这样的判断总是依赖于羸弱的标准，而是因为它可以把斗争揭露出来，以帮助你恢复对自身的主权。然后我们才开始谈自由。 以上是我们的纲领，还应该说说我们的现状。分裂影响着我思考的尺度，导致我总是关注极端琐碎或者极端宏大的问题，而它们都一样徒劳。要么是夸大个人心理的创伤以追求强烈的情感体验，要么是在漫长的历史、广袤的空间以及熙攘的人群中消解自身。疾病与治疗的道理，你自己病入膏肓，所以需要把自己完全摧毁来治病，但实际上既没有疾病也没有治愈。这就是现状。“人类的苦难可怕而又神秘，但正是它给了我们尊严。”你要停止用如同警察一般的目光来审视自身。可是我们又该如何面对个人和宏大？说起个人和宏大，我们好像是在说一个最底层与一个最高层的系统，我们失落了的处理方式就像我们的自然语言一样，穿梭其中，充满循环与自指。穿梭意味着主动、清醒的进入与退缩。

用户你说句话啊

随着我在github上摸爬滚打的经验增加，现阶段我信息技术成长方面的主要矛盾已经转变为日益增长的命令语句使用需要以及我不娴熟的命令语句使用技巧之间的矛盾。命令语句有种在和电脑说话的感觉，用起来很方便，习惯了以后感觉以前依赖图形界面的操作方式像是《百年孤独》所说：“世界新生伊始，许多事物还没有名字，提到的时候尚需用手指指点点。” Windows Terminal（WT） Windows Terminal颜值比cmd高出不少，而且有点图形界面，更加易用，装些脚本用着还是很舒服的。WT中常用的shell是window powershell,关于Terminal，shell等等用词之间的区别可以参考https://www.zhihu.com/question/21711307/answer/2231006377。不过WT的一个麻烦之处在于安装很多环境的时候不会自动添加Path变量，要手动添加一下才能用，可以用$env:path查看环境变量。 windows上可以用winget管理安装包，但我感觉没什么必要，因为windows有分区，本来用两三级就可以写出来的程序路径一用winget就变得老长，反而麻烦。自己装一下python，node.js和git也就基本够用了。 在使用git clone时可能会遇到git clone: error setting certificate verify locations这样的错误提示，此时只要设置git config --global http.sslverify "false"跳过ssl认证就好了，不过这个解决方案有点鸵鸟，有空还是应该把证书文件更新一下。有的时候在pip install中还会弹出关于ssl的警告，十有八九是代理的问题，关了就好。 windows上可以写bat批处理文件来自动执行一系列命令，比如我为了一键同步git仓库写了一个git.bat文件 1 2 3 4 5 6 7 @echo off cd C:\Users\qipao\Desktop\myblog git pull git cmp refine cd C:\Users\qipao\Desktop\tiddlywiki git pull git cmp refine shell 的使用的时候要注意提示符，参考http://c.biancheng.net/view/734.html Terminal on Mac Mac上自带了终端，颜值大概介于cmd和WT之间吧。Homebrew很好用，因为我本来就没心情去管mac的文件系统，正好homebrew帮我包办代替了，不过有的时候还是不得不找些文件，mac的文件目录可以参考https://www.cnblogs.com/gujiande/p/9447006.html。我的mac默认使用zsh shell，配置文件在用户的家目录下，通过command+shift+.查看隐藏文件就能看到，通常修改.zshrc文件。一些常用命令可以通过alias来快速输入，句法是alias A=B，注意在使用shell的时候不要随便加空格。 homebrew常用的命令是brew install和brew info，大部分包如果安上之后有问题，可以brew info看一下简介，比如装java过后要设定一个路径，在brew info java里提供了可以直接照抄的代码。 mac上可以用.command文件做批处理，比如这个git.command 1 2 3 4 5 6 7 8 9 cd /Users/a_big_apple/Desktop/myblog git pull git cm "refine" git push cd /Users/a_big_apple/Desktop/tiddlywiki git pull git cm "refine" git push exit 之后要用chmod +x filename赋可执行权限。...
