随着我在github上摸爬滚打的经验增加，现阶段我信息技术成长方面的主要矛盾已经转变为日益增长的命令语句使用需要以及我不娴熟的命令语句使用技巧之间的矛盾。命令语句有种在和电脑说话的感觉，用起来很方便，习惯了以后感觉以前依赖图形界面的操作方式像是《百年孤独》所说：“世界新生伊始，许多事物还没有名字，提到的时候尚需用手指指点点。”

Windows Terminal（WT）#

Windows Terminal颜值比cmd高出不少，而且有点图形界面，更加易用，装些脚本用着还是很舒服的。WT中常用的shell是window powershell,关于Terminal，shell等等用词之间的区别可以参考https://www.zhihu.com/question/21711307/answer/2231006377。不过WT的一个麻烦之处在于安装很多环境的时候不会自动添加Path变量，要手动添加一下才能用，可以用$env:path查看环境变量。

windows上可以用winget管理安装包，但我感觉没什么必要，因为windows有分区，本来用两三级就可以写出来的程序路径一用winget就变得老长，反而麻烦。自己装一下python，node.js和git也就基本够用了。

在使用git clone时可能会遇到git clone: error setting certificate verify locations这样的错误提示，此时只要设置git config --global http.sslverify "false"跳过ssl认证就好了，不过这个解决方案有点鸵鸟，有空还是应该把证书文件更新一下。有的时候在pip install中还会弹出关于ssl的警告，十有八九是代理的问题，关了就好。

windows上可以写bat批处理文件来自动执行一系列命令，比如我为了一键同步git仓库写了一个git.bat文件

Terminal on Mac#

Mac上自带了终端，颜值大概介于cmd和WT之间吧。Homebrew很好用，因为我本来就没心情去管mac的文件系统，正好homebrew帮我包办代替了，不过有的时候还是不得不找些文件，mac的文件目录可以参考https://www.cnblogs.com/gujiande/p/9447006.html。我的mac默认使用zsh shell，配置文件在用户的家目录下，通过command+shift+.查看隐藏文件就能看到，通常修改.zshrc文件。一些常用命令可以通过alias来快速输入，句法是alias A=B，注意在使用shell的时候不要随便加空格。

homebrew常用的命令是brew install和brew info，大部分包如果安上之后有问题，可以brew info看一下简介，比如装java过后要设定一个路径，在brew info java里提供了可以直接照抄的代码。

命令行工具#

支持图形化界面的程序数目是相当有限的，用上命令行之后有许多实用的工具，罗列如下：

搞到OpenAI API Key之后可以通过各种项目折腾ChatGPT。github项目https://github.com/npiv/chatblade允许在WT中调用chatGPT，用着挺顺手，不用上openai的网页了。

github上还有自动制作多语种对照epub书籍的https://github.com/yihong0618/bilingual_book_maker，也相当惊艳。不过调用Chatgpt的时候记得不要频繁换IP，否则API Key容易出问题。

ocrmypdf可以给pdf做文字识别，作为开源工具当然是免费的。链接在https://github.com/ocrmypdf/OCRmyPDF 。注意简体中文的ISO 639-2代码是chi_sim。

SSH与putty#

好男儿志在四方，租一个海外VPS还是很重要的，搬瓦工(Bandwagon)的VPS也不贵，而用服务器就免不了和SSH打交道。SSH(Secure Shell Protocol)是用来和远程服务器交互的协议，在Window上可以用Putty作为SSH终端，还可以借助Putty里的pscp与服务器传输文件。服务器配置是个有点复杂的问题，在《老鸨之王》中有详细介绍。Putty在连接服务器输入密码时复制似乎经常出问题。

Docker#

装在服务器上的docker没有GUI，需要用命令行操作。如果嫌部署命令太长，可以额外安一个Docker-compose，注意最新版本和有些服务器（比如CentOS7）的操作系统不兼容。远程访问docker容器需要在部署的时候给它分配一个外部端口。Docker最复杂的是网络，同一个IP地址上几个容器相互依赖容易出问题，有些程序可以直接在Vercel上部署以节省内存。
